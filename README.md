# 梦境 SVN 管理器

这是一个给游戏策划使用的 SVN 简化工具，目标是把常用操作做成少量中文按钮，降低误操作。

## 软件更新

- 软件启动时会检查 GitHub 最新 Release。
- 如果发现新版本，状态栏会显示“工具有新版本”。
- 点击状态栏提示，或点击“更多操作 -> 检查工具更新”，会打开更新面板。
- 更新面板会显示当前版本、GitHub 最新版本、更新说明和下载文件。
- 点击“下载并更新”后，软件会下载 Release zip，关闭当前程序，覆盖当前目录，然后自动重启。
- 开发环境下如果运行目录能找到 `.git`，仍支持通过 `git pull` 更新源码。

发布新版本时需要：

1. 修改 `SVNManager/SVNManager.csproj` 里的 `Version` / `AssemblyVersion` / `FileVersion` / `InformationalVersion`。
2. 运行 `.\Publish-Release.ps1`，脚本会用 Release 配置生成 Windows x64 发布目录和 zip。
3. 在 GitHub 创建新的 Release，例如 `v0.1.1`。
4. 把 zip 作为 Release 附件上传，文件名建议包含 `win-x64`，例如 `DreamSVNManager-win-x64-v0.1.1.zip`。

用户不需要每 10 分钟检查。软件只在启动时自动检查一次；之后用户可以手动点“检查工具更新”。

## 外部工具

- 发布包不再内置分久必合，避免每次更新都下载很大的 `ExternalTools`。
- 每台电脑第一次使用外部表格对比/合并前，需要在“更多操作 -> 设置”里选择本机的 `分久必合.exe`。
- 路径会保存到 `%AppData%\SVNManager\settings.json`，之后不用重复配置。
- 如果不配置分久必合，内置文本差异和 Excel 差异预览仍然可用，只是不能调用外部合并工具。

## 原生差异视图

- Excel/XML 差异使用 WinForms 原生统一视图展示，重点显示 ID、字段和差异内容。
- 差异内容会在同一单元格里显示旧值和新值，并用红绿颜色和字符级高亮标出变化片段。
- Excel/XML 差异支持按 ID、字段、内容筛选，并可一键复制当前筛选结果摘要。
- Excel/XML 支持内置三方合并：BASE、本地工作副本、远端 HEAD 自动合并；冲突项进入原生表格手动选择本地或远端。
- 内置表格合并会自动识别前几行里的字段行，并优先用可唯一定位的 `id` / `key` / `name` / `type` 等组合字段匹配行；无法安全识别时才退回单元格坐标。
- 文本差异支持双栏对比和统一视图，新增/删除行会用红绿底色和局部字符高亮辅助阅读。
- 文件树和历史改动文件都支持“当前本地 vs 远端 HEAD”对比，用来判断服务器最新版本和本地文件差异。
- 历史页 Diff preview 可以一键最大化，方便在有限窗口空间里审查具体文件差异。

## 原生现代化界面

- 主界面隐藏老式 Tab 页签，改用左侧自绘导航栏切换配置、改动、冲突、文件和历史。
- 历史列表和文件历史窗口使用卡片流展示提交，突出版本号、作者、时间和改动数量。
- 代码树和改动文件树使用无虚线 TreeView、自绘箭头、状态色图标和全行高亮。
- 提交窗口改为左右分栏：左侧写提交说明并显示字数，右侧用树形结构勾选待提交文件。
- 全部文件树改为懒加载，展开目录时才读取子目录，避免打开大工作副本时长时间卡顿。
- 更多操作里可以锁定/重置布局、管理常用目录收藏、打开最近操作时间线。

## 提交与更新安全

- 提交前会检查冲突、缺失文件、未加入文件、混合版本和远端 HEAD 是否领先。
- 存在阻断项时会直接阻止提交；存在风险项时会给出确认提示，避免误把脏状态提交出去。
- 历史刷新会优先读取远端 HEAD 到旧版本的日志，所以不拉取最新也能看到服务器最近提交。
- 内置表格三方合并写入前会把当前本地文件备份到 `%AppData%\SVNManager\merge-backups`，避免合并结果不满意时无法恢复。
- 写入 SpreadsheetML XML 时会刷新表格行列计数，并移除被写入单元格上的 `ss:Formula` / `ss:ArrayRange`，避免公式重新计算覆盖合并结果。

## 打开没反应时

如果双击 `SVNManager.exe` 没有任何反应，请让对方双击 `启动-带诊断.cmd`。

诊断脚本会：

1. 检查是否完整解压。
2. 启动 `SVNManager.exe`。
3. 如果程序立刻退出，显示 `%AppData%\SVNManager\startup.log` 里的错误。

把诊断窗口截图或 `startup.log` 发回来，就能定位是系统版本、文件缺失、权限拦截还是程序异常。
